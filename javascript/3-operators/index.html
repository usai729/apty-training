Javascript Operators

Topics

Operators are used to perform operations on variables and values in JavaScript. They can be classified into several types.

1. Arithmetic Operators

Arithmetic operators are used to perform mathematical operations like addition, subtraction, multiplication, etc.

  • +: Addition. Adds two values.
  • -: Subtraction. Subtracts one value from another.
  • *: Multiplication. Multiplies two values.
  • /: Division. Divides one value by another.
  • %: Modulus. Returns the remainder of a division operation.
  • ++: Increment. Increases a variable's value by 1.
  • --: Decrement. Decreases a variable's value by 1.
2. Assignment Operators

Assignment operators are used to assign values to variables.

  • =: Assignment. Assigns the right-hand value to the left-hand variable.
  • +=: Addition Assignment. Adds the right-hand value to the left-hand variable.
  • -=: Subtraction Assignment. Subtracts the right-hand value from the left-hand variable.
  • *=: Multiplication Assignment. Multiplies the left-hand variable by the right-hand value.
  • /=: Division Assignment. Divides the left-hand variable by the right-hand value.
  • %=: Modulus Assignment. Takes the modulus of the left-hand variable by the right-hand value.
3. Comparison Operators

Comparison operators are used to compare two values and return a boolean result (true or false).

  • ==: Equal to. Checks if two values are equal.
  • ===: Strict Equal to. Checks if two values are equal and of the same type.
  • !=: Not equal to. Checks if two values are not equal.
  • !==: Strict Not equal to. Checks if two values are not equal or not of the same type.
  • >: Greater than. Checks if the left value is greater than the right value.
  • <: Less than. Checks if the left value is less than the right value.
  • >=: Greater than or equal to. Checks if the left value is greater than or equal to the right value.
  • <=: Less than or equal to. Checks if the left value is less than or equal to the right value.
4. Logical Operators

Logical operators are used to combine multiple conditions in control flow statements.

  • &&: Logical AND. Returns true if both conditions are true.
  • ||: Logical OR. Returns true if at least one of the conditions is true.
  • !: Logical NOT. Reverses the boolean value of a condition (true becomes false, and false becomes true).
5. Ternary Operator

The ternary operator is a shorthand for an if-else statement. It takes three operands.

Syntax: condition ? expr1 : expr2

If the condition is true, expr1 is executed, otherwise, expr2 is executed.

Example: let result = (age >= 18) ? "Adult" : "Minor";

6. Type Operators

Type operators are used to check or convert the type of a variable or value.

  • typeof: Returns the type of a variable.
  • instanceof: Checks if an object is an instance of a specified class or constructor function.
7. Bitwise Operators

Bitwise operators perform operations on the binary representations of integers.

  • &: AND. Performs a bitwise AND operation.
  • |: OR. Performs a bitwise OR operation.
  • ^: XOR. Performs a bitwise XOR operation.
  • ~: NOT. Performs a bitwise NOT operation (flips bits).
  • <<: Left shift. Shifts the bits to the left.
  • >>: Right shift. Shifts the bits to the right.
  • >>>: Unsigned right shift. Shifts the bits to the right without preserving the sign.
8. Unary Operators

Unary operators operate on a single operand to return a result.

  • ++: Unary increment. Increases a variable’s value by 1.
  • --: Unary decrement. Decreases a variable’s value by 1.
  • typeof: Returns the type of a variable.
  • delete: Deletes an object property or array element.
  • void: Evaluates an expression and returns undefined.
9. Spread and Rest Operator

Both the spread and rest operators use the ... syntax but are used in different contexts.

  • ...: Spread operator. Used to expand elements into individual items (arrays or objects).
  • ...: Rest operator. Collects multiple elements into an array when used in function arguments or destructuring.
Operator Precedence in JavaScript

Operator precedence determines the order in which operators are evaluated in an expression. When there are multiple operators in an expression, JavaScript follows certain rules to evaluate them, starting with the operators that have higher precedence. Understanding operator precedence helps avoid unexpected results in calculations and ensures the desired order of operations.

1. Parentheses ()

Highest Precedence: Expressions inside parentheses are evaluated first, regardless of the operators outside.

let result = (2 + 3) * 4;  // (2 + 3) is evaluated first, then multiplied by 4.
console.log(result); // 20

2. Unary Operators

Increment (++) and Decrement (--) Operators: These operators have a higher precedence than binary operators. They can be applied as prefix or postfix. 

let a = 5;
let result = ++a + 3;  // Pre-increment: a is incremented first, then added to 3.
console.log(result); // 9

Unary + and - Operators: These operators are used to convert a value into a number (positive or negative).

let a = '10';
console.log(+a); // 10 (string '10' is converted to number)

Logical NOT (!): Negates the truthiness of a value.

let isTrue = false;
console.log(!isTrue); // true

3. Multiplication, Division, Modulus *

Next in Precedence: These operators are evaluated after parentheses and unary operators.

let result = 3 + 5 * 2;  // Multiplication happens first, then addition
console.log(result); // 13

4. Addition and Subtraction + -

Evaluated After Multiplication/Division: These operators have lower precedence than multiplication and division.

let result = 5 + 3 - 2;  // Addition and subtraction from left to right
console.log(result); // 6

5. Relational and Comparison Operators (<, >, <=, >=, ==, ===, !=, !==)

Used for Comparison: These operators are used to compare two values, returning true or false.

let a = 5;
let b = 10;
console.log(a < b);  // true
console.log(a === b); // false

6. Logical Operators (&&, ||)

Logical AND (&&): Evaluated before logical OR (||).

let a = true;
let b = false;
console.log(a && b); // false
console.log(a || b); // true

7. Assignment Operators (=, +=, -=, *=, /=, etc.)

Lower Precedence: These operators are evaluated after all the operators listed above.

let a = 5;
a += 3;  // a = a + 3;  // Assignment happens after the addition
console.log(a); // 8

8. Conditional (Ternary) Operator ? :

Evaluated After Logical Operators: The ternary operator (condition ? expr1 : expr2) has a higher precedence than assignment but lower than most arithmetic and relational operators.

let a = 5, b = 10;
let result = (a < b) ? 'a is less than b' : 'a is not less than b';
console.log(result); // 'a is less than b'

9. Comma Operator ,

Lowest Precedence: The comma operator allows you to evaluate multiple expressions in a single statement, returning the result of the last expression.

let result = (2 + 3, 4 + 5);
console.log(result); // 9 (last expression is evaluated)

How to Manage Precedence in Complex Expressions

Use Parentheses: Parentheses should be used to clearly indicate the desired evaluation order in complex expressions.

let result = (3 + 5) * (2 - 1); // Without parentheses, the addition would happen first, then multiplication.
console.log(result); // 8

Know the Precedence: Always be aware of operator precedence when writing expressions to avoid unintended results.
